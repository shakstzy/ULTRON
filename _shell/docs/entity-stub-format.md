# Entity Stub Format

Canonical format for every file under `_global/entities/<type>/<slug>.md`. The global tier is identity only; workspace-specific synthesis stays in each workspace's own `wiki/entities/`.

## Required frontmatter

```yaml
---
slug: <kebab-case>
type: <person | company | project | place | concept | program | venue | other>
canonical_name: <full name as it appears in conversation>
aliases: []                              # alternate names / nicknames / former slugs
created_at: <YYYY-MM-DD>
last_promoted_from: <workspace-slug>     # which workspace triggered promotion
status: active                           # active | archived (never deleted)
---
```

Optional identifier block (omit absent fields):

```yaml
identifiers:
  email: []                # canonical lowercase
  phone: []                # E.164 (`+15125551234`)
  domain: []               # for companies / projects
  github: <handle>
  linkedin: <slug>
  twitter: <handle>
  url: <canonical-url>
```

## Required sections

### `## Identity`

One paragraph. Stable cross-workspace facts: who / what / why exists. No workspace-specific synthesis. No deal terms, no opinions, no gossip.

### `## Workspace synthesis`

Auto-generated bullet list, one bullet per workspace where this entity has a workspace-level page:

```md
- `eclipse` — [page](../../workspaces/eclipse/wiki/entities/people/<slug>.md): one-line context
- `personal` — [page](../../workspaces/personal/wiki/entities/people/<slug>.md): one-line context
```

Lint pass populates and refreshes this section. Manual edits get overwritten.

### `## Backlinks`

Auto-generated by `_shell/bin/build-backlinks.py`. Lists every place in any workspace that wikilinks `[[@<slug>]]`. Manual edits get overwritten.

## Promotion criteria

An entity gets promoted from a workspace's `wiki/entities/` to `_global/entities/` when ALL of:

1. It appears in 2+ workspaces' `wiki/entities/`, OR is referenced via `[[@slug]]` from 2+ workspaces.
2. Its identity is stable across workspaces (same human / company / project, not a name collision).
3. The audit agent flagged it as a promotion candidate ≥ 7 days ago in `_meta/learning-proposals.md`.

Promotion is one-directional. Adithya runs the `promote-entity` skill (`_shell/skills/promote-entity/`) which:

1. Creates `_global/entities/<type>/<slug>.md` from a merged-and-cleaned version of the workspace pages.
2. Leaves the original workspace pages in place; they now contain workspace-specific synthesis only, and reference the global stub via `[[@slug]]`.
3. Logs the promotion to `_global/entities/_promotion-log.md`.

## What does NOT go in global entity stubs

- Workspace-specific deal terms, equity, dollar amounts (stay in workspace `wiki/entities/` or `raw/`)
- Opinions, gossip, secrets (stay in workspace pages — global is for stable identity only)
- Dated event details (go in workspace synthesis pages)
- Anything from a workspace flagged `exclude_from_super_graphify: true` (currently: `onlyfans`)
- Children / family of subscribers, fans, dating-platform matches whose real identity should not cross workspaces

## Aliases (rename without losing identity)

If a slug needs to change (canonical name correction, slug typo fix), use the `alias` skill:

```
/alias <old-slug> <new-slug>
```

Creates a redirect stub at the old slug pointing to the new, rewrites every `[[@old-slug]]` reference across all workspaces. Never hand-edit aliases — the skill ensures every wikilink gets fixed.

## Deletion

Entity stubs are NOT deleted. To retire one: set `status: archived` in frontmatter, add a one-line note in `## Identity`. Lint pass keeps archived stubs out of the super-graphify pass but leaves them readable. If an entity must truly disappear (legal request, etc.), Adithya runs `_shell/bin/rename-slug.py --hard-delete <slug>` which is `CONFIRM`-gated.
