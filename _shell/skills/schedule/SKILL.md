# schedule

Source-of-truth: `_shell/config/global-schedule.yaml` plus each workspace's `config/schedule.yaml`. Compiles those into launchd plists at `_shell/plists/`. Loads them via `launchctl bootstrap` only on explicit request.

## Modes

### `compile [--dry-run]`
Reads `_shell/config/global-schedule.yaml` plus every `workspaces/<ws>/config/schedule.yaml`. Generates plists at `_shell/plists/com.adithya.ultron.<job>.plist`. Reports orphans (plist files at `_shell/plists/` that don't match any schedule.yaml entry).

### `load <plist-name | --all>`
`launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/<label>.plist`. First symlinks the plist from `_shell/plists/` to `~/Library/LaunchAgents/` if not already linked. Prints `launchctl list | grep com.adithya.ultron` afterward.

### `unload <plist-name | --all>`
`launchctl bootout gui/$(id -u)/<label>` and removes the symlink from `~/Library/LaunchAgents/`. Plist file at `_shell/plists/` is preserved.

### `status`
Prints which plists exist at `_shell/plists/`, which are loaded (per `launchctl list`), and last run time per job (parsed from `_logs/<label>.out.log`).

### `remove <plist-name>`
Unloads + deletes the plist file from `_shell/plists/` AND the symlink. Use when retiring a job permanently.

## Plist generation rules

Per `(account, source)` ingest plist (NOT per workspace):
- Label: `com.adithya.ultron.ingest-<source>-<account-slug>`
- Schedule: from `sources.<source>.cron` in any workspace's `schedule.yaml` (if multiple workspaces specify different crons for the same `(source, account)`, the **most frequent** wins).
- Command: `flock -n /tmp/<label>.lock $ULTRON_ROOT/_shell/bin/run-stage.sh ingest-source <source> <account>`.

Per-workspace lint / graphify / audit plists:
- Label: `com.adithya.ultron.<job>-<workspace>`
- Schedule: from `workspace_jobs.<job>.cron` in `workspaces/<ws>/config/schedule.yaml`.
- Command: `flock + run-stage.sh <job> <workspace>`.

Cross-workspace plists (audit, weekly-review, graphify-supermerge, apple-contacts-sync, ledger-compact):
- Label: `com.adithya.ultron.<job>`
- Schedule: from `_shell/config/global-schedule.yaml`.

Every plist includes:
- `StandardOutPath` / `StandardErrorPath` → `_logs/<label>.{out,err}.log`.
- `EnvironmentVariables`: `PATH=/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin`, `ULTRON_ROOT=<absolute>`, `HOME=<absolute>`.
- `WorkingDirectory` → ULTRON root.
- `RunAtLoad: false`.

## Cron string compilation

The `cron:` field uses standard 5-field cron (`<min> <hour> <dom> <mon> <dow>`). The compile mode handles these patterns:
- `M H * * *` → daily at H:M.
- `M H-H2 * * *` → hourly within range.
- `M */N * * *` → every N hours.
- `M H * * <0-6>` → weekly at H:M on weekday.
- `M H D * *` → monthly at H:M on day-of-month.
- `M H D */N *` → every N months at H:M on day-of-month (compiled to multiple intervals).

Each pattern compiles to one or more `<dict>` entries in `StartCalendarInterval`. Patterns the compiler can't translate are reported and skipped.

## Self-review (~150 tokens)

- `plutil -lint` passes for every generated plist.
- For `compile`: every entry in any `schedule.yaml` produced exactly one plist; orphans listed.
- For `load`: `launchctl list | grep com.adithya.ultron` shows the new label.
- For `unload`: the label is gone from `launchctl list`.
- No plist was loaded that wasn't named in the request (no implicit loads).
- For `status`: every plist is reported with `loaded` + `last_run`.

## Hard rules

- NEVER auto-load on `compile`. The user explicitly invokes `load` after review.
- NEVER edit plists in `~/Library/LaunchAgents/` directly. Always edit the source at `_shell/plists/` (regenerated by `compile`) and run `unload` then `load` to refresh.
- NEVER call `launchctl reboot` or anything that affects other users' agents.
- Loading and unloading require explicit user confirmation in chat — `load --all` is destructive (it boots ~10 agents into the user's session).
