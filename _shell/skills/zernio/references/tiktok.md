# TikTok

`POST /v1/posts`. Mandatory `creator-info` precheck before every post. TikTok fields live in **`tiktokSettings` (top-level)**, NOT in `platformSpecificData` — this differs from IG/YT/Twitter.

## Account prerequisites

OAuth permissions: `video.publish`, `video.upload`. Unaudited API clients have varying posting restrictions; always check `creator-info` first.

## Mandatory precheck

```
cli.sh creator-info <accountId> [video|photo]
```

Response shape:
```json
{
  "privacyLevels": ["PUBLIC_TO_EVERYONE", "FOLLOWER_OF_CREATOR", "MUTUAL_FOLLOW_FRIENDS", "SELF_ONLY"],
  "postingLimits": { "used": 2, "cap": 10, "resetAt": "..." },
  "commercialContentTypes": ["none", "brand_organic", "brand_content"]
}
```

The caller's `privacy_level` MUST be in the returned `privacyLevels`. If `PUBLIC_TO_EVERYONE` is missing for an unaudited client, surface that — `SELF_ONLY` lands the post privately, never publicly viewable.

## Request shape

```json
{
  "content": "Caption. Videos: max 2200 chars. Photo carousels: max 4000 chars.",
  "mediaItems": [{ "type": "video", "url": "https://..." }],
  "platforms": [{ "platform": "tiktok", "accountId": "<accountId>" }],
  "tiktokSettings": {
    "privacy_level": "PUBLIC_TO_EVERYONE",
    "allow_comment": true,
    "allow_duet": true,
    "allow_stitch": true,
    "content_preview_confirmed": true,
    "express_consent_given": true,
    "video_made_with_ai": false,
    "video_cover_timestamp_ms": 1000,
    "video_cover_image_url": "https://...",
    "commercialContentType": "none",
    "draft": false
  },
  "publishNow": true
}
```

## `tiktokSettings` fields

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `privacy_level` | enum | YES | Must be in `creator-info.privacyLevels` |
| `allow_comment` | bool | YES | |
| `allow_duet` | bool | videos | |
| `allow_stitch` | bool | videos | |
| `content_preview_confirmed` | bool | YES (legal) | TikTok requires this explicit flag |
| `express_consent_given` | bool | YES (legal) | TikTok requires this explicit flag |
| `video_made_with_ai` | bool | no | AIGC label |
| `video_cover_timestamp_ms` | number | no | Default 1000ms |
| `video_cover_image_url` | string | no | JPG/PNG/WebP, max 20MB |
| `media_type` | string | no | `photo` for carousels |
| `photo_cover_index` | number | no | 0-based carousel cover |
| `description` | string | no | Caption for photo carousels |
| `auto_add_music` | bool | no | Photo carousels only |
| `commercialContentType` | enum | no | `none | brand_organic | brand_content` |
| `draft` | bool | no | Send to Creator Inbox instead of publishing |

## `draft` and `publishNow` precedence

When `draft: true`, the skill MUST also set `publishNow: false` in the top-level payload. `draft: true` with `publishNow: true` is contradictory; Zernio behavior is undefined. Print both values in the gate so the caller can verify.

## Limits

| Item | Value |
|------|-------|
| Video | 1 per post; MP4/MOV/WebM; 3s–10min; 4 GB; 9:16 only viable |
| Photo carousel | up to 35 images; JPEG/PNG/WebP; 20 MB each |
| Mixing | cannot mix video + image in one post |
| Caption | videos 2,200 chars; photo carousels 4,000 chars |

## Errors

| Error | Class | Note |
|-------|-------|------|
| `Invalid privacy level` | fatal_content | Value not in creator-info response (caller bug) |
| `30001` | fatal_content | Invalid codec |
| `30005` | retryable_later | Processing timeout |
| `daily posting limit` | retryable_later | Third-party API daily cap |
| `Content flagged for spam/moderation risk` | fatal_content | Moderation rejection |

## Unsupported

Music/sound library (except `auto_add_music` for photo carousels), creating duets/stitches, going live, effects, editing post-publication, comments/DMs, FYP analytics.

## AI disclosure

`video_made_with_ai: true` if any of: media generated by Sora/Runway/Kling/Veo, audio TTS or voice-cloned, realistic likeness altered. TikTok integrated C2PA detection in Jan 2025 — non-disclosure when metadata is present is treated as intentional and issues an immediate strike.
