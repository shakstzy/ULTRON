# Format: apple-contacts (SPECIAL — direct to global)

## File granularity
One markdown file per contact, in `_global/entities/people/<slug>.md`. NOT in any workspace's `raw/`.

## Slug derivation (priority)
Same as iMessage:
1. Apple Contacts full name → kebab-case ASCII, max 40 chars.
2. Email local-part + domain stem.
3. E.164 phone (`+15125551234` → `phone-15125551234`).
4. Hash-prefix fallback.

Once chosen, stable forever — recorded in the file's frontmatter.

## Frontmatter

```yaml
---
# Universal envelope (REQUIRED)
source: apple-contacts
workspace: _global
ingested_at: <ISO 8601>
ingest_version: 1
content_hash: <blake3 of frontmatter+body>
provider_modified_at: <Apple Contacts modificationDate>

# Apple Contacts specific
title: Sydney Hayes
slug: sydney-hayes
type: person
canonical_uri: lifeos:_global/entities/people/sydney-hayes
aliases: ["Syd"]
identifiers:
  email: ["sydney@eclipse.audio", "sydney.hayes@gmail.com"]
  phone: ["+15125551234"]
  slack: []                          # populated by slack ingest if available
last_synced: <ISO 8601>
global: true
---
```

## Body

```markdown
## Notes

(populated by user / wiki agent — contacts-sync NEVER touches body content
after first creation)

## Backlinks

(auto-generated by build-backlinks.py — DO NOT EDIT MANUALLY)
```

## Behavior
- Frontmatter is fully OVERWRITTEN on each sync (Contacts is source of truth).
- Body is preserved if manually edited (skill never touches body after first creation).
- First-time creation: body has placeholder `## Notes` for the user / wiki agent.
- Slug is recorded in frontmatter `slug:`. Once written, never changes.

## Pre-filter
- Skip contacts with no email AND no phone AND no name (empty record).
- Skip contacts marked as deleted in Contacts (their record won't return from the AppleScript query).

## Dedup key
`apple-contacts:<slug>` — the file path itself IS the dedup. No `ingested.jsonl` row needed.
