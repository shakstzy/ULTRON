## AI Notes

### AI Agent Assurance Platform Concept

- Core problem: Companies racing to deploy AI agents without reliable pre-production validation
- Current eval tools only provide risk scores (e.g. “82% accurate”) - useless when remaining 18% can cause compliance violations, lawsuits, financial damage
- Fundamental issue: AI evaluating AI is like “students grading their own exams”

### Proposed Solution Architecture

- Agent “gyms” - sandboxed environments mirroring real workflows
- Deterministic verification using parsers, compilers, ledgers, calculations vs. opinion-based scoring
- Counterexample traces showing exact failure points

    - Example: Agent correctly sums 2+3 but writes result in A4 instead of A5
- Multi-agent tribunal system for subjective cases

    - Advocate agent, critic agent, judge agent
    - Claims 15-25% improvement in eval efficacy
- Vertical-specific judge models trained on company/industry data

### Competitive Landscape & Challenges

- Vigil AI: Focuses on runtime monitoring, policy enforcement, incident detection
- Brain Trust: MLOps platform for model evaluation
- Vellum: Programmatic checks requiring manual metric definition
- Key differentiators claimed:

    - Pre-production proofs vs. post-incident detection
    - Autonomous metric generation vs. manual definition
    - Vertical-specific fine-tuned judges vs. general models

### Critical Feasibility Issues Raised

- Ground truth data collection extremely expensive/difficult

    - Would need crowdsourced data partnerships initially
    - Requires deep product knowledge to define meaningful evaluations
- Tribunal system still relies on foundational LLMs (Claude, GPT-4) initially

    - Same bias issues as current LLM-as-judge approaches
    - Need 6+ months of customer data to build vertical models
- Customer data sharing challenges

    - Companies unlikely to share proprietary training data
    - Would require “army of people in India” to manually create datasets
- Building library of verticalized models per application extremely resource-intensive
